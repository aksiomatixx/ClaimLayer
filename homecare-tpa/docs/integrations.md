# Integration Specifications

All external API credentials are stored in environment variables. Never hardcode credentials. Never commit `.env` files. See `.env.example` in the root for required variable names.

All integrations require a signed HIPAA Business Associate Agreement before production use.

## 1. FileHandler Enterprise (JW Software)

**Purpose:** Authoritative financial ledger and claims management system. Every financial transaction, reserve change, document, and diary lives here. DWC audits pull from FileHandler. This is the system of record.

**Credential type:** API key (provided by JW Software after account setup)  
**Base URL:** `https://api.jwsoftware.com/filehandler/v1` (confirm with JW Software)  
**Auth header:** `Authorization: Bearer {FILEHANDLER_API_KEY}`

### Error Handling
- Retry on 429 (rate limit) and 5xx with exponential backoff: 1s, 2s, 4s
- On 401: refresh API key (alert admin if key expired)
- On 404: claim not found — do not create duplicate, alert admin
- Log every call: endpoint, status code, latency, claim ID

## 2. ADP Workforce Now

**Purpose:** Pull employee demographics and pay statements on claim creation. Calculate average weekly wage (AWW) and California TD rate.

**Credential type:** OAuth2 Client Credentials  
**Auth URL:** `https://accounts.adp.com/auth/oauth/v2/token`  
**API base:** `https://api.adp.com`  
**Required scopes:** `openid profile hr-confidential payroll`  
**Note:** ADP requires a practitioner certificate for TPA entities. Apply at marketplace.adp.com.

### Auth Flow
```http
POST https://accounts.adp.com/auth/oauth/v2/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id={ADP_CLIENT_ID}
&client_secret={ADP_CLIENT_SECRET}

→ { "access_token": "...", "expires_in": 3600, "token_type": "Bearer" }
```
Cache the token. Refresh when `expires_in - 60 seconds` elapsed.

### Employee Demographics
```http

Authorization: Bearer {access_token}

→ {
    "workers": [{
      "associateOID": "aoid-bc-001",
      "person": {
        "legalName": { "givenName": "Maria", "familyName": "Santos" },
        "birthDate": "1981-03-15",
        "homeAddress": {
          "lineOne": "1842 W 7th St",
          "cityName": "Los Angeles",
          "countrySubdivisionLevel1": { "codeValue": "CA" },
          "postalCode": "90057"
        }
      },
      "businessCommunication": {
        "landlines": [{ "formattedNumber": "(213) 555-0142" }]
      },
      "workerDates": { "originalHireDate": "2019-01-14" },
      "businessCommunication": { "jobTitle": "Home Health Aide II" }
    }]
  }
```

### Pay Statements (last 26 pay periods for AWW)
```http
GET https://api.adp.com/payroll/v1/workers/{associateOID}/pay-statements
  ?$top=26
Authorization: Bearer {access_token}

→ Array of pay statements with: grossPay, regularHours, otPay, payPeriodStartDate, payPeriodEndDate
```

### AWW / TD Rate Calculation
```python
def calculate_td_rate(pay_statements: list) -> dict:
    # California LC §4453 — Average Weekly Wage
    total_gross = sum(p.gross_pay for p in pay_statements)
    weeks_worked = len(pay_statements)
    aww = total_gross / weeks_worked

    # California 2026 TD rates
    TD_MIN = 252.03   # Updated Jan 1, 2026
    TD_MAX = 1680.29  # Updated Jan 1, 2026

    td_rate = aww * (2/3)
    td_rate = max(TD_MIN, min(TD_MAX, td_rate))

    return { "aww": round(aww, 2), "td_rate": round(td_rate, 2) }
```

## 3. Manifest MedEx (QHIO / California DxF)

**Contact:** DxFDirectory@manifestmedex.org  
**Note:** HomeCare TPA must sign the CalHHS DxF Data Sharing Agreement (DSA) before accessing. Apply at dxf.chhs.ca.gov.

## 4. Anthropic Claude API

**Purpose:** Core AI engine. Compensability analysis, reserve recommendations, RFA-vs-MTUS evaluation, clinical document summarization, diary generation, notice drafting.

**Model:** `claude-sonnet-4-20250514` (always use this exact string — do not use `claude-sonnet-4` without version)  
**Auth:** `x-api-key: {ANTHROPIC_API_KEY}` header  
**Endpoint:** `POST https://api.anthropic.com/v1/messages`

### Standard Request Structure
```javascript
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": process.env.ANTHROPIC_API_KEY,
    "anthropic-version": "2023-06-01"
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1500,
    system: systemPrompt,
    messages: [{ role: "user", content: userPrompt }]
  })
});

const data = await response.json();
const text = data.content?.find(b => b.type === "text")?.text || "{}";
const result = JSON.parse(text.replace(/```json|```/g, "").trim());
```

### Always request JSON output
Every Claude call in this system should request structured JSON. Include in every system prompt:
```
Respond ONLY with valid JSON. No markdown, no backticks, no preamble, no explanation outside the JSON structure.
```

### Key Prompts (stored in `backend/prompts/`)

**`compensability_analysis.txt`** — Initial claim analysis  
**`reserve_recommendation.txt`** — Reserve setting based on injury type and clinical facts  
**`rfa_mtus_evaluation.txt`** — RFA vs MTUS guidelines evaluation  
**`document_summary.txt`** — Clinical document summarization (PR-2, IMR, QME reports)  
**`diary_generation.txt`** — Generate diary set from claim state  
**`notice_draft.txt`** — Draft statutory notice text  

> Store prompts as text files, not hardcoded strings. This allows updating prompts without code changes.

### Rate Limits and Error Handling
- Claude API rate limits vary by tier. Start with Tier 1, upgrade as volume grows.
- On 529 (overloaded): retry after 30 seconds, max 3 retries.
- On 400 (invalid request): log the prompt + error, create admin alert. Do not retry.
- Never expose raw Claude responses to end users. Always parse and validate JSON before use.
- If JSON parsing fails: log the raw response, create a manual review task, do not auto-approve anything.

## 6. Lob.com

**Purpose:** Print and mail physical statutory notices. USPS first-class delivery. Required for: DWC-7, TD benefit notices, delay notices, denial letters, DWC-9 payment notices.

**Auth:** HTTP Basic auth with API key as username, empty password  
**Base URL:** `https://api.lob.com/v1`

### Send a Letter
```http
POST https://api.lob.com/v1/letters
Authorization: Basic {base64(LOB_API_KEY + ":")}
Content-Type: application/json

{
  "description": "DWC-7 Notice of Representation — HHW-2026-041",
  "to": {
    "name": "Maria Santos",
    "address_line1": "1842 W 7th St",
    "address_city": "Los Angeles",
    "address_state": "CA",
    "address_zip": "90057",
    "address_country": "US"
  },
  "from": {
    "name": "HomeCare TPA",
    "address_line1": "{our_address}",
    "address_city": "{our_city}",
    "address_state": "CA",
    "address_zip": "{our_zip}",
    "address_country": "US"
  },
  "file": "data:application/pdf;base64,{base64_pdf}",
  "color": false,
  "double_sided": false,
  "mail_type": "usps_first_class",
  "metadata": {
    "claim_id": "HHW-2026-041",
    "notice_type": "DWC7",
```

## Integration Testing

Each integration has a sandbox environment:

| Integration | Sandbox | Notes |
|---|---|---|
| FileHandler | Sandbox credentials from JW Software | Separate environment, safe to create test claims |
| ADP | ADP sandbox | Test employee IDs provided by ADP |
| Manifest MedEx | Test environment | Contact MedEx for test org credentials |
| Health Gorilla | Sandbox | Test patient data available |
| Enlyte | UAT environment | Contact Enlyte for UAT credentials |
| Lob.com | Test API key (prefix `test_`) | No physical mail sent in test mode |
| Anthropic | Same API key | Sandbox = staging environment, real API calls |
| SendGrid | Same API key | Use test email addresses only |
| Twilio | Test credentials | No real SMS sent in test mode |

All staging environment API calls must use sandbox credentials. The `.env.staging` file must never contain production credentials.
